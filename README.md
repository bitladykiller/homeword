# 小葵的污染体检报告

> 以「向日葵小葵」为主角的环境科普静态网站。项目把空气、水体、土壤污染对植物的影响组织成一份可浏览、可交互的"体检报告"，引导用户走完「污染观察 → 伤害诊断 → 修复行动」的完整认知链路。

## 项目定位

- **角色驱动科普**：将抽象的环境污染数据转化为一株向日葵的生命体征（叶片、根系、光合作用），降低理解门槛。
- **展区式叙事**：模拟博物馆展览动线，首页为序厅，三个展区层层递进，每个展区下设 3 个专题子页面。
- **纯静态实现**：无框架、无构建、无后端，HTML + CSS + 原生 JS 即可完整运行。
- **交互增强理解**：点击观察点、步骤切换、标签页、卡片翻转、康复进度条、绿色行动卡生成等多种交互形式嵌入内容页面。

## 页面详解

### 首页（index.html）

- **Hero 区**：标题 + 副标题 + 半健康半枯萎的向日葵主视觉图
- **体检摘要卡片**：生命状态、叶片、根系、光合、疑似原因，5 张卡片快速交代小葵当前状况
- **三大展区导航**：带编号的展区卡片，引导用户按顺序进入

### 第一展区：污染环境记录

| 页面 | 内容要点 |
|------|---------|
| `air.html` | 点击叶片上的 4 个观察点（叶尖焦枯、斑点、黄化、灰尘覆盖），查看空气污染对叶片的直接伤害 |
| `water.html` | 分步骤展示受污染水体从排放口到植物根部的传输路径 |
| `soil.html` | 可交互的土壤剖面，点击不同土层查看污染分布 |

### 第二展区：植物伤害观察

| 页面 | 内容要点 |
|------|---------|
| `plant-damage.html` | 小葵体检示意图，点击叶片 / 根系 / 光合按钮高亮对应体检报告项 |
| `leaf.html` | 健康叶片与污染叶片的状态对比切换 |
| `root.html` | 点击根系观察点，查看地下损伤细节 |
| `photosynthesis.html` | 光反应与暗反应受损机制说明 |

### 第三展区：生态修复建议

| 页面 | 内容要点 |
|------|---------|
| `control.html` | 空气、水体、土壤治理方案三标签切换 |
| `phytoremediation.html` | 植物修复卡片翻转交互，查看不同植物的修复能力 |
| `action.html` | **核心交互页**（详见下方） |

## 核心特色：康复交互系统（action.html）

这是项目中交互最密集、实现最精细的页面。

### SVG 矢量小葵

使用纯 SVG 绘制的角色化向日葵，包含：

- **花瓣**：背面 16 片 + 正面 16 片，使用径向渐变 + 旋转矩阵精确排布
- **花心**：径向渐变面部 + 种子环
- **面部细节**：眉毛、眼睛（含高光）、嘴巴、腮红
- **茎叶**：渐变茎干 + 4 片带叶脉的叶片（左右各两片高低错落）
- **特效**：柔光阴影滤镜（feDropShadow）、花瓣纹理（feTurbulence + feColorMatrix）、光晕圆环、星星点缀
- **土壤**：底部泥土 + 小石子

### 多阶段康复状态

通过容器元素上的 `data-healed` 属性（0~6）驱动 CSS 状态切换：

| data-healed | 勾选数 | 视觉表现 |
|-------------|--------|---------|
| 0 | 0 项 | 灰暗色花瓣，叶片萎缩，进度环 0%，表情疲惫 |
| 1~2 | 1~2 项 | 花瓣颜色转黄，叶片小幅舒展，单部位状态改善 |
| 3~4 | 3~4 项 | 花瓣金黄，叶片展开，多个部位恢复正常 |
| 5 | 5 项 | 花朵放大并缓慢摇摆动画，进度环接近满 |
| 6 | 6 项（全选） | 全面康复：花瓣饱满金黄带发光效果，进度环 360° 全满带呼吸光晕，表情开心 |

### 进度环

基于 CSS `conic-gradient` 实现，使用 `@property` 注册自定义属性实现渐变角度平滑过渡动画。

### 康复部位映射

每个环保行动关联一个康复部位（`data-heal` 属性）：

| 行动 | 康复部位 |
|------|---------|
| 少用一次性塑料制品 | 叶片（土壤改善 → 叶） |
| 节约用水 | 根系（水源保护 → 根） |
| 垃圾分类 | 叶片（土壤改善 → 叶） |
| 低碳出行 | 光合（空气净化 → 光） |
| 不随意倾倒废液 | 根系（水源保护 → 根） |
| 参与校园绿植养护 | 光合（空气净化 → 光） |

### 绿色行动卡

提交表单后展示个性化承诺卡片，列出用户选择的所有行动。

## 技术实现

### 文件清单

```
sunflower-pollution-report/
├── index.html
├── pollution-record.html
├── air.html
├── water.html
├── soil.html
├── plant-damage.html
├── leaf.html
├── root.html
├── photosynthesis.html
├── repair.html
├── control.html
├── phytoremediation.html
├── action.html                  ← 含完整 SVG 小葵插画
├── css/
│   └── style.css                ← 全站样式 + SVG 状态动画
├── js/
│   └── main.js                  ← 全站交互脚本
├── assets/
│   └── images/                  ← PNG 角色图、场景图、图标
├── package.json
└── README.md
```

### 技术要点

- **CSS 变量**：定义 `--primary-green`、`--sunflower-yellow`、`--alert-red` 等全局色彩
- **`conic-gradient`**：进度环实现
- **`@property`**：注册自定义属性使渐变角度可动画化
- **SVG 滤镜**：`feTurbulence` 生成花瓣纹理，`feDropShadow` 添加柔光阴影
- **`data-healed` 属性选择器**：无 JS 状态管理参与样式切换，CSS 侧通过 `#actionContainer[data-healed="N"]` 控制各阶段表现
- **原生 JS**：`updateHealingState()` 读取 checkbox 选中数，更新 `data-healed` 属性、状态文字、进度百分比和提示语

## 快速开始

### 直接打开

双击 `index.html` 即可在浏览器中预览。

### 本地服务器（推荐）

```bash
cd sunflower-pollution-report
python3 -m http.server 8000
```

访问 `http://localhost:8000`

> 部分浏览器在 `file://` 协议下可能限制资源加载，推荐使用本地服务器。

## 设计规范

| 维度 | 说明 |
|------|------|
| 主色 | 生态绿 `#2F6B4F`、向日葵黄 `#F4C430`、警示红 `#D9534F` |
